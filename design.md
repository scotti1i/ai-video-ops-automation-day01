# Design Spec v6 · 视频闭环生产平台

> 状态：`approved`
> 确认人：Scott（2026-07-15 直接指令："全面对标 Resend 的风格和 taste"）
> 日期：2026-07-15
> 本版取代 v5 的「视觉语言」与页面观感条款；v5 的主流程、脚本候选内容合同、创作设定合同、可见产物合同**继续全部生效**。本版新增视觉体系（Resend 对标）、列宽与信息密度修正、以及数据回流→裂变的提炼展示。

## 对标

- 正向对标：**Resend dashboard**（Scott 提供的 resend.com/emails 截图为准）。要借鉴的是：纯白画布、侧栏灰胶囊选中态、圆角搜索/筛选控件、表格外框圆角卡、状态软色 pill、克制到几乎无色的中性灰体系、大量呼吸感。
- 辅助对标：OpenRouter / APIMart 的"高级感、和谐感、只给想看的"。
- 反例：v5 时期的蓝灰 tinted 面板、13px 挤压表格、@homelab-… 截断列、批次行三列空白。

中性体系去蓝调、归真灰（chroma ≤ 0.006，保留 260 冷调残响）：

| Token | 值 | 用途 |

| `--surface` | `oklch(1 0 0)` | 卡片/弹层 |
| `--surface-muted` | `oklch(0.972 0.003 260)` | 表头底、hover、次级面板 |
| `--surface-selected` | `oklch(0.951 0.004 260)` | 侧栏选中胶囊、激活分段 |
| `--border` | `oklch(0.928 0.005 260)` | 默认边框 |
| `--border-strong` | `oklch(0.868 0.007 260)` | 控件边框 |

| `--text-soft` | `oklch(0.37 0.02 260)` | 副文字 |
| `--text-muted` | `oklch(0.548 0.016 260)` | 弱文字（白底对比度 ≥4.5:1，不许再淡） |

## 字体与字阶

`Inter / SF Pro Text / PingFang SC / system-ui` 单字体栈不变。产品 UI 固定 rem 阶（非流式）：

- 页标题 28/36 · 700 · `-0.01em`；页副题 14/22 · 400 · muted
- 区块标题 16/24 · 600；表头 13/20 · 500 · muted（正常大小写，不做全大写）
- 正文与表格主体 **14**/21 · 400（告别 13px 挤压）；辅助行 13/18；最小字号地板 **12px**（废除全部 `text-[11px]`）
- 数字一律 `tabular-nums`

## 组件规格（`@fifty/workbench-ui`，50 天全体共用）

- **按钮**：高 40（`sm` 36）、`rounded-[10px]`。primary = `--brand` 实底白字，hover `--brand-strong`；secondary = 白底 `--border-strong` 边，hover `--surface-muted`；quiet = 幽灵。全部 2px `--focus` 焦点环（offset 2）。同屏只有一个 primary。
- **状态 pill（Badge）**：`rounded-full`、软色底+同系深文字、无边框、高 24、`px-2.5`、13px·500。语义只有五种：成功绿、等待橙、失败红、演示/信息蓝、中性灰。
- **输入/选择器**：高 40、`rounded-[10px]`、白底、`--border-strong` 边、placeholder 用 `--text-muted`（不许更淡）。搜索框带内嵌放大镜，宽度舒展（工作台 ≥ 360px）。
- **Tabs**：分段胶囊式（灰轨道内白/灰 pill，Resend 的 Sending/Receiving 样式）为默认；underline 变体保留给抽屉。
- **表格（data-table）**：整表包在 `rounded-[14px]` 1px 边框卡内；表头透明底 muted 文字；行高≈56–64（`py-3.5`）；`divide-y --border`；hover 整行 `--surface-muted`；行首实体图标放 32px `rounded-[8px]` 描边方容器（Resend 邮件图标样式）。
- **侧栏（app-shell）**：与内容同白，`border-r` 分隔；导航项高 36、`rounded-lg`、图标 16 + 13.5px·500 文字；选中 = `--surface-selected` 胶囊 + `--text`；未选中 muted，hover 灰底。
- **抽屉/弹窗**：24px 内边距起步，标题 18/26 · 600；只用一级阴影。抽屉宽 `min(100vw, 640px)`——脚本正文与逐镜分镜要能并排读，420px 是不够的。
- **抽屉结构（硬约定）**：标题栏常驻不随正文滚动（关闭按钮永远够得着）；正文独立纵向滚动，横向**永不溢出**。长文件名、长链接一类不可断行的内容一律折行（`break-all`），禁止用 `truncate` 的 `nowrap` 去赌容器宽度——它会经由 grid/flex 的 `min-width:auto` 把整列撑宽、顶破抽屉。
- **成片预览**：短视频以竖屏为主，不锁 16:9。按成片自身比例显示并限高（`max-h-[min(60vh,520px)]`），不铺满宽度、不留黑边。

## Day-01 页面级修正（本版新增的硬要求）

1. **工作台列宽重分配**（1440 基准）：批次/脚本 弹性占大头；账号 `w-44`（头像+可读 handle，截断必须有 title）；进度 `w-56`；结果 `w-32` 两行数字；下一步 `w-28`。任何列不允许出现"全列相同截断值"（如一排 @homelab-…）。
2. **批次父行不许空三列**：账号列 = 本批账号头像堆叠（≤3 + "+N"）；进度列 = 阶段分布（"待脚本 1 · 待成片 1"从下一步列移过来）；结果列 = 本批最佳数据（"最高 46万 播放"）或 "—"；下一步 = 单一主动作（继续选稿 N / 生成脚本 / 上传成片）。
3. **候选默认不勾选**：用户必须主动比较后勾选；"只选结构通过"保留为快捷动作；主按钮在 0 选中时禁用并显示"先勾选要测试的候选"。
4. **裂变弹窗带依据**：弹窗顶部显示"父视频表现"提炼块（来自后端 `performance_brief`：对比同批倍数、订单转化、高赞评论诉求），用户不再盲写"本轮变化"。
5. **发布日历**：视频列以标题为主行、编号为 muted 尾注；失败行操作是"处理 →"而不是"等待执行"；状态 pill 用统一语义色。
6. **矛盾空态**：分镜缺失提示只在"有脚本无分镜"时出现；无脚本时只显示"脚本待生成"一个空态。
7. **路由**：监听 `hashchange`，浏览器前进/后退可用。
8. 侧栏、顶栏、S/01 彩蛋等 v5 结构条款照旧。

## 三个禁止项（v6）

1. **禁止蓝色铺陈**：海军蓝只出现在主按钮、当前态、焦点环；不做蓝 tinted 面板/底色铺底，中性层一律真灰。
2. **禁止两个方向的假高级**：不加渐变/玻璃/阴影堆料，也不许把表格再挤回 11–13px 的 Excel 密度；高级感只来自留白、层级、真实数据。
3. **禁止组件词汇分叉**：同一动作全站同形态；状态一律 pill，不混用文字色块/描边徽章/圆点三种写法。

（v5 的脚本候选三禁令、创作设定三禁令继续生效。）

## 验收标准（v6 增量）

- 1440×960 截图与 Resend Emails 页并排看，属于同一档次的克制与呼吸感；无蓝灰 tinted 面板残留。
- 工作台批次行五列全部有内容；展开后账号列每行可分辨是哪个账号。
- 候选清单在不展开的情况下，10 条的**开头两句互不相同**（内容合同，由生成侧保证，界面按原文展示）。
- 键盘走完 开批→候选→选稿→裂变 全程焦点可见。
- `uv run video-ops check` 全绿；真实页面逐页复核后才算关闭。
