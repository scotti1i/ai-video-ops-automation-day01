# Design Spec v8 · 画布手感对标 LiblibAI

> 状态：`已实现`
> 日期：2026-07-16
> 本版**只改画布外壳的观感与手感**（v7 的"画布即产品/节点=阶段/判断在面板/砍掉四入口"全部继续生效，面板内部不动）。
> 起因：v7 把画布做成了静态信息图，和 LiblibAI / n8n / ComfyUI 的"活画布"手感差距大。方向：对标 LiblibAI 的画布手感，拓扑锁死。

## 对标与边界

- **对标**：LiblibAI 工作流画布（= ComfyUI 派系的节点画布约定）。抄的是**通用功能范式**：可拖可缩的点阵画布、节点卡带端口圆点、加粗贝塞尔连线、小地图、缩放控件、悬停/选中态。
- **不抄**：LiblibAI 的 logo、品牌色、自定义图标、插画、CSS/代码（专有 + 许可陷阱）。配色继续用我们自己的 Resend 墨蓝体系（design.md v6），字体不变。
- **拓扑锁死**（v7 铁律不变）：节点不可增删、连线不可重连、无右键加节点菜单。我们抄的是 liblib 的**手感**，不是它的**可编辑性**。

## 从"静态图"到"活画布"——逐项改

### 1. 画布本体

- **平移**：`panOnDrag` 开（空白处拖动画布）。
- **缩放**：`zoomOnScroll` + `zoomOnPinch` 开，`minZoom 0.4 / maxZoom 1.5`；缩放锚点跟随光标。
- **点阵背景**：`<Background variant="dots" />`，点用中性灰、随画布平移移动——这是"无限画布"手感的来源。
- **进入即 fitView**：首次把整个环铺进视口，之后用户自由平移缩放；不再每次 resize 都强制归位（那是 v7 的静态做法，删掉 always-fit 的 ResizeObserver）。
- **缩放控件**：左下角自绘一组克制的控件（放大 / 缩小 / 适配全部），不用 React Flow 默认 `<Controls />`（太重、样式不搭）。
- **小地图**：右下角 `<MiniMap />`，节点用状态色的小块。虽然只有 8 个节点用不太上，但它是 liblib 手感的一部分——**放，但做小、做淡**。

### 2. 节点卡片解剖（ComfyUI/liblib 的招牌长相）

每个节点从"白盒子+角标"升级成**带头部的卡片**：

```
┌─────────────────────────────┐
○ │ [图标] 选稿          [19] │ ○   ← 头部：阶段图标色块 + 名字 + 计数徽章
  │ ● 等你                     │     ← 状态行：状态圆点 + 文字（等你/自动/出问题）
└─────────────────────────────┘
左端口                     右端口
```

- **端口圆点**：左侧输入、右侧输出，各一个 `<Handle>`。**视觉锚点，非交互**——`isConnectable={false}`，连线从它出入，但用户拖不出新线（拓扑锁死）。颜色随状态。
- **阶段图标**：每个阶段一个 lucide 图标塞进头部左侧的 6px 圆角色块（这批想做什么=闪电/生成候选=魔法棒/选稿=清单/成片=胶片/发布=纸飞机/数据回流=波形/提炼=奖杯/再做一版=分叉）。图标是我们自己的 lucide，不是 liblib 的。
- **计数徽章**：头部右侧，tabular-nums；0 时节点整体转灰、徽章隐藏。
- **状态**：三种，圆点+文字。等你=品牌墨蓝、自动=灰、出问题=红。卡片描边跟随（等你=墨蓝、出问题=红、其余=默认灰）。
- 卡片：白底、`rounded-[12px]`、1px 边框、悬停时轻微上浮 + 阴影加深（`shadow-overlay` 的更轻档）。

### 3. 连线

- **贝塞尔曲线**（`type="default"` 即 bezier，不再用 v7 的直线/step）。
- **加粗**：`stroke-width 2`，颜色中性灰；箭头 `MarkerType.ArrowClosed`。
- **流动动效**：只在"环在转"的语义上给一条 `animated` 虚线流动（从 数据回流→提炼→再做一版→起点 这段回边），其余静态。不给全部连线都加流光（那是廉价科技风）。

### 4. 交互手感

- **节点可拖 + 记住位置**：拖动节点位置持久化到 `localStorage`（按工作区 id）。**只动坐标，不动拓扑**。给一个"恢复默认布局"的小入口（缩放控件旁）。
- **悬停**：卡片上浮 2px + 阴影，端口点变亮。
- **选中**：`elementsSelectable` 开，选中卡片描边加粗为品牌墨蓝 + ring；但选中不触发面板（点击才开面板，区分单击开面板 vs 框选）。→ 若单击=开面板 与 单击=选中 冲突，以**开面板**为准，选中态仅作 hover 视觉，不做多选。
- **键盘**：Tab 走节点、Enter 开面板、Esc 关面板；焦点环保留。

## 手机 390

画布退化为纵向卡片列表（v7 已有），但卡片升级成与桌面同构的解剖（图标+名字+计数+状态圆点），去掉端口与连线，用竖直连接线串起来表达顺序。不做双指缩放。

## 三个禁止项

1. **禁止把锁死的画布做出"可编辑"的错觉之外，也禁止让端口看起来像坏的**：端口是视觉锚点，光标移上去不给"可拖拽"的抓手指针（`cursor: default`），避免用户拖了发现连不动而觉得是 bug。
2. **禁止 liblib 的皮**：不抄它的品牌色（那个荧光绿）、logo、图标、深色主题。我们是墨蓝 + 纸白 Resend 体系，只借它的**结构与手感**。
3. **禁止流光泛滥 / 装饰科技风**：只有回边一条流动虚线表达"闭环在转"，其余克制。不发光、不玻璃、不渐变。

## 验收标准

- 并排放 LiblibAI 画布截图旁边，**结构与手感是同一档**：可拖可缩、点阵动、节点带端口、连线是粗贝塞尔、有小地图和缩放控件。
- 但**一眼能看出这不是 liblib**：配色是墨蓝纸白不是荧光绿、干净克制。
- 拖动节点、缩放画布、平移——**真机 60fps 顺滑**（这次真有 pan/zoom 了，React 19 手感必须真机验，不能只看间接证据）。
- 拓扑锁死：拖不出新连线、加不了节点、右键无加节点菜单。
- 点节点仍然开对应面板（v7 交互不变）。
- 1440 / 390 双视口真机逐页验收。
