# Design Spec v7 · 画布即产品

> 状态：`已实现`
> 日期：2026-07-15
> 本版取代 v6 的**页面结构**（工作台表格、侧栏四入口、发布日历页）。
> v6 的**视觉语言**（Resend 色板、字体、圆角、动效时长）与 v5 的**内容合同**（脚本候选五问、质量硬门、诚实纪律）**继续全部生效**。

## 为什么推翻 v6 的页面结构

v6 把一条流水线画成了一张表：批次折叠、一行九个点击目标、"下一步"列、进度三段、发布日历独立成页。结果是定规格的人自己都“非常 confuse”。

诊断不是像素问题，是**形状问题**：

- 它按"批次"组织，但批次在生成候选后五分钟就不重要了；每天真正的问题是"现在该干什么"。
- "脚本"一词在同一列里有三种含义（批次内容 / 正式视频产物 / 待选候选）——数据模型漏进了界面。
- 一行九个点击目标、六个去处，建立不了肌肉记忆。每次点击前的那一秒犹豫，就是 confuse。

**这个东西本质就是一条流水线。流水线的界面就该是流水线本身，不是流水线的台账。**

## 对标与选型

- **形状对标**：n8n / ComfyUI / Dify 的节点画布——一眼看懂流程走向，不用教程。
- **视觉对标**：仍是 Resend（v6 色板与字体原样继承）。借 n8n 的**形状**，不借它的**皮肤**：不要深色科技风、不要发光连线、不要节点上堆图标。
- **画布库**：`@xyflow/react` v12.11.2 · **MIT** · peer `react >=17`（我们 19.2.7 兼容）。**画布这个轮子不自己写。**
- **不做的事**：不部署 n8n / Dify 服务。后端仍是我们自己的 FastAPI + SQLite，`video-ops demo` 一条命令本地起。画布只是前端库。

## 页面结构：一屏 = 一个环

首页就是整条流水线，**没有侧栏，画布即导航**：

```text
  ┌──────────┐    ┌──────────┐    ┌─────────┐    ┌────────┐    ┌────────┐
  │ 这批想做  │───▶│ 生成候选  │───▶│  选稿   │───▶│  成片  │───▶│  发布  │
  │  什么     │    │   AI     │    │ ⏸ 等你  │    │ ⏸ 等你 │    │  自动  │
  └──────────┘    └──────────┘    └─────────┘    └────────┘    └────────┘
       ▲                                                             │
       │                                                             ▼
       │          ┌──────────────┐         ┌──────────┐      ┌──────────┐
       └──────────│ 裂变 ⏸ 等你  │◀────────│ 提炼 自动 │◀─────│ 数据回流 │
      带着赢家基因 └──────────────┘  这条赢了 └──────────┘  自动 └──────────┘
        开下一批                      因为…
```

**闭环第一次肉眼可见。** 这一屏就是拍视频时的分镜。

### 节点长什么样

每个节点只有四件事：

1. **人话名字**——「选稿」不是「候选评审」，「再做一版」不是「裂变」（术语见下）。
2. **计数徽章**——这个阶段现在有几条。0 时节点静默（灰、无徽章）。
3. **状态**——只有三种：`等你`（品牌深色描边 + 徽章）· `自动`（灰）· `出问题`（红）。
4. **一个动作**——点节点 = 打开右侧面板。没有第二个点击目标。

### 判断在面板里，不在画布上

点节点 → 右侧面板（复用 v6 已修好的 640px 抽屉）：

| 节点 | 面板里做什么 |
|---|---|
| 这批想做什么 | 商品、Context、参考、条数、创作设定（原"开始一批视频"弹窗） |
| 生成候选 | 只读：这批依据 + 生成进度 |
| 选稿 | 比较 10 条候选、展开读全文、编辑、重写、勾选进成片 |
| 成片 | 待成片清单，逐条上传/登记，看竖屏预览 |
| 发布 | 账号选择、排期、发布记录、失败处理（**吃掉原"账号"页和"发布日历"页**） |
| 数据回流 | 各发布任务的指标快照与评论 |
| 提炼 | 这一轮谁赢了、赢在哪个变量（performance_brief） |
| 裂变 | 带着赢家基因开下一批 |

**画布上永远不出现口播正文。** 内容判断是面板的事。

## 砍掉什么

- **视频工作台表格**（批次折叠、九个点击目标、"下一步"列、进度三段）
- **侧栏四入口**：账号 / 商品 / 发布日历 三页并入对应节点面板；视频列表降级为画布右上角的搜索
- **"批次"作为一级浏览维度**——它只在「选稿」面板里存在（同一批天然处在同一阶段，画布免费保留了分组）

保留：视频详情抽屉（从面板里打开单条）、全部后端、全部内容合同。

## 术语改人话

| 现在 | 改成 |
|---|---|
| Context | 这批想做什么 |
| 裂变 | 再做一版（赢家基因） |
| 候选 / 待选稿 | 待选脚本 |
| 结构通过 | 可以拍了 |
| 需修改 | 还差证据 |
| 待观察 | 数据还没起来 |

## 三个禁止项

1. **禁止把画布做成通用工作流编辑器。** 流水线是固定的：节点不可增删、连线不可重连、不做条件分支编辑器。我们不是在做 n8n，我们是在用画布表达一个已知的流程。用户要改流程 → 那是另一个产品。
2. **禁止把内容判断塞进画布。** 读脚本、看成片、挑稿子一律在面板里。画布只回答"现在该干什么"，不回答"这条写得好不好"。
3. **禁止节点装饰。** 不要发光连线、流光动画、节点上的图标墙、深色科技风。动效只表达两件事：有东西流过去了（150–220ms）、这个节点在等你（静态描边，不闪）。

## 验收标准

- **5 秒**看懂这是个什么流程，不需要任何说明文字。
- **一眼**看出现在该干什么：等你的节点唯一高亮，其余安静。
- 从「数据回流」能一路看回「生成候选」——**闭环在一屏内闭合**，1440 不用滚动。
- 12 条视频和 500 条视频，**画布长得一样**（只有徽章数字变），不会被数据撑爆。
- 键盘可达：Tab 依次走节点，Enter 开面板，Esc 关；焦点可见。
- 画布不是死图：某条视频推进后，对应节点徽章实时变化。
- 390 手机：画布退化为纵向的阶段列表（同样的节点、同样的顺序、同样的一个动作），不做双指缩放画布。
- 真实页面双视口逐页验收——自动测试通过只是工程门，Scott 的真实页面验收才关产品门。
