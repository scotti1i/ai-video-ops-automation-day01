# 真实模型验证 · 2026-07-14

## 范围

使用一段来自本项目真实业务定义、已经脱敏且不含账号、客户、媒体路径或密钥的 Context，验证工作模式下的模型生产链。密钥、地址和模型名均从本地安全配置注入，没有写入数据库、日志或仓库。

## 产品内路径

```text
POST /api/videos
→ POST /api/videos/{video_id}/generate (producer=openai)
→ ScriptArtifact v1
→ StoryboardArtifact v1
```

## 结果

- API 成功保存 1 个脚本版本和 1 个分镜版本。
- 产物来源为 `model`，分镜包含 8 个可编辑镜头。
- 生成器记录为兼容聊天 JSON 回退；模型正文仅保存在忽略目录的本地验证数据库。
- 相同结果合同已由单元测试覆盖 Responses、结构化聊天、SSE JSON 和有限 Schema 中继。

## 真实限制与修复

首次调用暴露了三个真实兼容差异：中继地址只提供域名根、Responses 尚未实现、聊天接口默认返回 SSE 且不接受带 `$defs/$ref` 的结构化 Schema。适配器现在会补标准 `/v1` 前缀，按端点能力降级，并最终用本地 `GeneratedPlan` 合同严格校验；认证、限流和网络超时不会盲目触发多次降级调用。

这份证据只证明脚本/分镜真实模型链路，不代表任何视频生成、平台发布或转化效果。
